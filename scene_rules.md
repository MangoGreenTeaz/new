代码中的场景规则以优先级顺序依次执行：

1. 先判断高优先级场景
2. 仅对当前 `scene_label` 仍为空字符串的记录继续判断下一个场景
3. 一旦某条记录被命中并赋值，后续场景不再覆盖该记录

## 规则维护约定

后续每新增或修改一个场景时，需要同步修改两个地方：

1. `scene_label.py`
   - 在 `build_scene_rules()` 中新增或修改规则

   - 更新下方规则表
   - 确保文档描述与代码逻辑一致

## 实现原则

- `scene_label` 默认值为空字符串 `""`
- 所有场景按优先级从小到大执行
- 后续场景只能处理尚未标注的记录
- 场景扫描过程中一旦遇到 `scene_label` 已非空的记录，会立刻截断当前候选范围，不跨越已标注数据继续扩展
- 数据按 `udid`、`time` 排序后处理，场景判断严格限制在同一个用户内
- 分块处理时会缓存每个块尾部最后一个 `udid` 的数据，等待和下一块拼接后再统一判断，避免同一用户被切断
- 若新增场景依赖新的输入字段，需要同步更新输入字段说明文档
- 若调整场景优先级，需要同步更新本文件中的规则表

## 已实现场景说明

### 高铁场景

#### 场景范围识别

1. 在同一个 `udid` 内按时间顺序扫描
2. 遇到 `poi` 包含 `高铁站` 的首条记录，作为候选范围起点，并纳入范围
3. 从起点继续向后扫描，只要持续出现“高铁相关特征”，就继续扩展范围
4. 若连续 3 条记录都不含高铁相关特征，则停止扫描
5. 若扫描过程中遇到 `hours_since_prev > 12` 的记录，则立即停止扫描，并将该条前一条作为当前高铁范围可结束的位置
6. 候选范围终点取“最后一条仍然具有高铁相关特征的记录”
7. 若扫描过程中遇到已被更高优先级场景标注的记录，则立即停止当前高铁候选范围扩展

#### 高铁相关特征

满足以下任一条件即视为高铁相关特征：

- `poi` 包含 `高铁站`
- `app_ticket == true`
- `move_fast == true`
- `move_cross_city == true`

#### 场景丢弃条件

若候选范围满足以下任一条件，则整个高铁场景范围丢弃，不进行标注：

- 范围内任意记录的 `poi` 包含 `机场`
- 范围内没有任何记录满足 `move_cross_city == true`
- 范围内不存在节点 B 或节点 C

#### 节点定义

- 节点 A：候选范围第一条记录，标注为 `抵达始发高铁站`
- 节点 B：候选范围内第一次出现 `move_fast == true` 的记录，标注为 `高铁行程途中`
- 节点 C：
  - 先取候选范围最后一条记录的 `city` 作为“最后一个城市”
  - 再从范围起点向后找第一次出现该城市的记录
  - 若该记录不在节点 B 之后，则继续向后找第一个位于节点 B 之后、且城市等于最后一个城市的记录
  - 该记录标注为 `抵达终点高铁站`

#### 区间标注规则

- 节点 A：`抵达始发高铁站`
- A 和 B 之间：`高铁站候车`
- 节点 B：`高铁行程途中`
- B 和 C 之间：`高铁行程途中`
- 节点 C：`抵达终点高铁站`
- C 之后直到场景范围最后一条：`离开终点高铁站`

### 自驾场景

#### 场景范围识别

1. 在同一个 `udid` 内按时间顺序扫描
2. 同时满足 `move_fast == true` 且 `move_cross_city == true` 的记录作为自驾锚点
3. 从锚点向前、向后分别扫描，只要持续出现“自驾相关特征”，就继续扩展范围
4. 若某个方向连续 3 条记录都不含自驾相关特征，则该方向停止扫描
5. 若某个方向扫描到的记录满足 `hours_since_prev > 3`，则该方向立即停止扩展，且该记录不纳入自驾范围
6. 范围起点和终点分别取该方向上最前/最后一条仍具有自驾相关特征的记录
7. 若任一方向扫描过程中遇到已被更高优先级场景标注的记录，则该方向立即停止扩展

#### 自驾相关特征

满足以下任一条件即视为自驾相关特征：

- `move_fast == true`
- `move_cross_city == true`
- `poi` 包含 `加油站`
- `poi` 包含 `电动车充电站`
- `poi` 包含 `服务区`
- `poi` 包含 `停车场`

#### 场景丢弃条件

若候选范围满足以下任一条件，则整个自驾场景范围丢弃，不进行标注：

- 范围内不存在任何 `move_cross_city == true` 的记录
- 范围内任意记录的 `poi` 包含 `高铁站`
- 范围内任意记录的 `poi` 包含 `机场`

#### 范围内标签优先级

对自驾范围内的每条记录按以下优先级依次判断：

1. 包含 `服务区` POI：`服务区休息`
2. 包含 `加油站` 或 `电动车充电站` POI：`驾驶途中加油、充电`
3. 包含 `停车场` POI：`去停车场停车/取车`
4. 其余记录：`自驾途中`

#### 范围最后一条特殊规则

- 若范围最后一条包含 `停车场` POI，则标记为 `去停车场停车/取车`
- 否则标记为 `驾车抵达终点`

### 服务区休息

#### 判定条件

- 当前记录未标注
- 当前记录的 `poi` 包含 `服务区`
- 同一用户内，当前记录前 3 条、当前记录本身、后 3 条这些实际存在的 7 条窗口中，至少有 1 条满足 `move_fast == true`

#### 标注结果

- 满足条件时，当前记录标注为 `服务区休息`
