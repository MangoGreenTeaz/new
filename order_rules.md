## 当前代码状态

- 脚本文件：`order.py`
- 输入文件默认值：`a_label.csv`（`scene_label.py` 的输出）
- 输出文件默认值：`a_order.csv`
- 输出列：`time`, `udid`, `text`, `scene_label`, `order`
- 随机种子：`42`（固定，保证可复现）
- 当前支持订单类型：高铁（火车）、飞机
- 分块大小：`100,000` 行

## 处理流程

1. 读取 `scene_label.py` 的输出文件
2. 按 `udid` 分组，保证同一用户的数据完整处理
3. 筛选出 `scene_label` 属于高铁或飞机场景的记录
4. 对筛选后的记录按场景类型变化和时间间隔切分为独立片段
5. 对每个片段生成订单字符串
6. 将订单字符串写入片段内所有记录的 `order` 列
7. 非高铁/飞机场景的记录 `order` 列为空字符串

## 场景分类

| scene_label | 订单类型 |
| --- | --- |
| `抵达始发高铁站` | 火车 |
| `高铁站候车` | 火车 |
| `高铁行程途中` | 火车 |
| `抵达终点高铁站` | 火车 |
| `离开终点高铁站` | 火车 |
| `抵达始发机场` | 飞机 |
| `机场内活动` | 飞机 |
| `飞机行程途中` | 飞机 |
| `抵达终点机场` | 飞机 |
| `离开终点机场` | 飞机 |
| 其他所有标签 | 不生成订单 |

## 片段切分规则

同一个 `udid` 内，具有订单标签的记录按以下条件切分为独立片段：

1. **场景类型变化**：从火车切换到飞机，或从飞机切换到火车
2. **时间间隔过大**：相邻记录的时间差超过 12 小时

每个片段代表一次独立的出行行程，生成一条独立的订单。

## 高铁订单生成规则

### 出发时间

按以下优先级确定出发时间：

1. 取**最后一条候车记录**与**第一条行程途中记录**的中间时刻
2. 若候车记录不存在，改为**最后一条抵达始发高铁站记录**
3. 若行程途中记录不存在，改为**第一条抵达终点高铁站记录**
4. 若两侧都能确定，取二者时间的中点
5. 若仅有一侧，取该侧时间
6. 若都不存在，取片段第一条记录的时间

### 到达时间

按以下优先级确定到达时间：

1. **抵达终点高铁站**第一条记录的时间 −1 分钟
2. 若不存在，取**离开终点高铁站**第一条记录的时间 −1 分钟
3. 若都不存在，取片段最后一条记录的时间

### 出发城市

按以下优先级确定出发城市：

1. 从**抵达始发高铁站**和**候车**记录中，按时间排序取第一个有效城市
2. 若不存在，取**第一条行程途中记录**的 `from_city` 或 `current_city`
3. 若都不存在，取片段第一条记录的 `current_city`

### 到达城市

按以下优先级确定到达城市：

1. 从**抵达终点高铁站**和**离开终点高铁站**记录中，按时间排序取第一个有效城市
2. 若不存在，取片段最后一条记录的 `current_city`

## 飞机订单生成规则

### 出发时间

按以下优先级确定出发时间：

1. **最后一条机场内活动记录**的时间 +1 分钟
2. 若不存在，取**最后一条抵达始发机场记录**的时间 +1 分钟
3. 若都不存在，取**第一条飞机行程途中记录**的时间 −1 分钟
4. 若全部不存在，取片段第一条记录的时间

### 到达时间

按以下优先级确定到达时间：

1. **抵达终点机场**第一条记录的时间 −1 分钟
2. 若不存在，取**离开终点机场**第一条记录的时间 −1 分钟
3. 若都不存在，取片段最后一条记录的时间

### 出发城市

按以下优先级确定出发城市：

1. 从**抵达始发机场**和**机场内活动**记录中，按时间排序取第一个有效城市
2. 若不存在，取**第一条飞机行程途中记录**的 `from_city` 或 `current_city`
3. 若都不存在，取片段第一条记录的 `current_city`

### 到达城市

按以下优先级确定到达城市：

1. 从**抵达终点机场**和**离开终点机场**记录中，按时间排序取第一个有效城市
2. 若不存在，取片段最后一条记录的 `current_city`

## 输出格式

订单字符串统一格式如下：

```
订单类型：火车，出发时间：2026/01/05 14:30，到达时间：2026/01/05 17:45，出发城市：上海市，到达城市：南京市
```

格式规范：

| 项目 | 规范 |
| --- | --- |
| 分隔符 | 全角逗号 `，` |
| 冒号 | 全角冒号 `：` |
| 时间精度 | 年/月/日 时:分（不含秒） |
| 日期与时间间距 | 单空格 |
| 订单类型取值 | `火车` 或 `飞机` |

### 写入规则

- 片段内每一行的 `order` 列都写入相同的完整订单字符串
- 非高铁/飞机场景的记录 `order` 列为空字符串

## 辅助字段提取

`order.py` 在处理过程中从 `text` 字段二次提取以下辅助信息：

| 字段 | 提取规则 | 用途 |
| --- | --- | --- |
| `current_city` | 优先使用输入中的 `city` 列；若无则从 `text` 中提取 `城市：` 与逗号之间的内容 | 出发/到达城市 |
| `from_city` | 从 `text` 中匹配 `前N分钟从XXX跨城市/市内` 提取出发城市 | 高铁/飞机行程途中的出发城市回退 |

## 随机偏移

- 出发时间和到达时间各加 ±5 分钟的随机偏移
- 使用固定随机种子 `42`，保证每次运行结果一致
- 若偏移后出发时间晚于到达时间，则交换二者

## 实现原则

- 使用 Polars 库分块读取，支持千万级数据量
- 分块处理时缓存每个块尾部最后一个 `udid` 的数据，等待和下一块拼接后再统一处理，避免同一用户被切断
- 所有时间解析使用 `%Y/%m/%d %H:%M` 格式
- 非订单场景记录原样保留，`order` 列为空字符串
